# Tool Box

这里用于沉淀可以直接复用的工具能力，而不是纯说明文档。

## 适合放什么

- 命令行工具使用入口
- Copilot 与 MCP 实战套路
- 常用脚本、模板与自动化片段
- 可视化辅助工具入口

## 当前已规划入口

- [MCP Explorer](/tools/mcp-explorer)：面向 MCP server、tool contract、调用路径与调试视图的统一入口。

## 建议的渐进式组织方式

1. 先从文档型工具目录开始，把现有高频操作套路整理进来。
2. 再逐步加入轻量前端工具页，例如 Prompt Formatter、JSON/YAML Helper、Workflow Snippet Viewer。
3. 当工具页数量变多，再把高交互工具拆到独立 React 子应用。

## 下一步建议

- 建一个 Prompt 工具页入口
- 建一个 MCP 调试工具入口
- 建一个常用命令与模板索引页
