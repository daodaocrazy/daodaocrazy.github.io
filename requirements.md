# 旅行路线规划页面 · 需求文档

## 1. 文档说明

本页面面向 GitHub Pages 静态站点，作为独立页面 `travel-route.html` 存在。目标是：
让用户能够在可拖拽 / 缩放的地图上交互地规划一条闭环旅行路线，
为每一段路线选择对应的交通方式，生成可被他人（或跨设备）加载查看的 JSON 数据文件。

文档覆盖功能需求、交互流程、数据结构、技术选型、边界与非目标。

## 2. 角色与使用场景

- 角色：个人旅行者 / 博客站点维护者
- 场景 1：在浏览器中规划一条包含多城市的闭环旅行路线，直观可视化。
- 场景 2：导出路线 JSON，提交到 GitHub Pages 仓库，用于长期保存和跨设备加载。
- 场景 3：在不同设备上导入 JSON 查看之前规划的路线，或作为旅行回顾。

## 3. 功能需求

### 3.1 地图与交互

- R1.1 使用开源轻量地图库 Leaflet（v1.9.4），并以 CDN 方式引入 CSS / JS，
  避免对 GitHub Pages 静态托管之外的构建依赖。
- R1.2 底图瓦片使用 OpenStreetMap（`https://{s}.tile.openstreetmap.org/{z}/{x}/{y}.png`），
  并在地图上展示 OpenStreetMap 的版权声明。
- R1.3 支持地图拖拽平移、滚轮缩放、加号 / 减号按钮缩放。
- R1.4 自动适应窗口尺寸变化，避免尺寸为 0 导致瓦片不显示（调用 `invalidateSize`）。
- R1.5 在移动端（宽度 ≤ 780px）切换为上下布局，上半部分为侧栏，下半部分为地图。

### 3.2 途经点管理

- R2.1 侧栏提供「添加途经点」按钮；点击后进入"添加模式"，地图光标切换为十字准星。
- R2.2 在"添加模式"下，用户点击地图上的任意位置即可新增一个途经点，
  弹出输入框让用户填写名称（可留空，自动使用 `途经点 N`）。
- R2.3 每个途经点在侧栏对应一张"卡片"，展示名称、坐标、操作按钮（重命名 / 上移 / 下移 / 删除）。
- R2.4 地图上的途经点以 `L.marker` 呈现，可拖动调整位置；拖动后坐标自动更新。
- R2.5 地图 `fitBounds` 自动将视图调整到当前所有途经点与路段所在的区域。
- R2.6 提供"清空"按钮，二次确认后删除全部途经点。

### 3.3 闭环路线与交通方式

- R3.1 路线按照侧栏中出现的顺序连接所有途经点，
  并将"最后一个点 → 第一个点"作为闭环返程段，形成闭合的旅行路线。
- R3.2 每一段路线（从点 i → 点 i+1）由用户在前一个点的卡片中选择交通方式。
- R3.3 支持以下交通方式及其视觉区分：
  - 飞机（红色虚线）
  - 公共交通（蓝色实线）
  - 自驾（绿色实线）
  - 步行（橙色细虚线）
  - 火车（紫色虚线）
  - 轮船（青色虚线）
- R3.4 悬停路线时，显示工具提示（Tooltip）包含：交通方式、是否闭环返程、本段距离（km）。
- R3.5 侧栏底部展示汇总信息：途经点数量、路段数量、总距离（km）。

### 3.4 数据导入 / 导出与同步

- R4.1 支持导出：点击"导出 JSON"将当前路线下载为 `travel-route-YYYY-MM-DD.json` 文件。
- R4.2 支持导入：点击"导入 JSON"可选择本地 `.json` 文件并加载；格式错误时给出明确提示。
- R4.3 页面打开时按以下优先级自动加载路线数据：
  1. **URL 参数优先**：若地址形如 `travel-route.html?data=data/my-trip.json`，从该路径（相对页面目录或绝对 URL）fetch JSON。
  2. **默认仓库路径**：自动尝试加载 `data/travel-route.json`（相对页面所在目录）。
  3. **本地缓存**：回退到浏览器 `localStorage` 中的最近一次数据。
  4. **示例数据**：三者都不可用时展示一条包含 4 个示例点的示例闭环。
- R4.4 侧栏顶部显示"当前数据来源"提示条，内容包括：
  - 来源标识（URL 参数 / 默认路径 / 本地缓存 / 本地文件 / 示例）
  - 指向源 JSON 文件的链接（可直接在新标签页查看）
  - URL 参数加载时额外提供"复制分享链接"按钮
- R4.5 提供"重新加载"按钮，按 R4.3 顺序再次 fetch 仓库中的 JSON。
- R4.6 用户需将导出的 JSON 文件手动加入 GitHub Pages 仓库的 `data/` 目录，文件名推荐为 `travel-route.json`；
  或将文件放在任意路径，然后通过 `?data=path/to/file.json` 参数分享给他人。
- R4.7 当默认路径的 JSON 未加载到（例如首次使用，仓库中尚未放置），提示条以温和颜色提示"尚未发现 data/travel-route.json，导出 JSON 后保存到此路径即可自动加载"。

### 3.5 可访问性与体验

- R5.1 所有交互（添加 / 删除 / 清空）均有二次确认或可撤销机会，避免误操作。
- R5.2 UI 使用中文，避免复杂技术用语；颜色对比度满足基础可读性。
- R5.3 页面顶部标题与侧栏提示说明如何使用。

## 4. 非目标（Out of Scope）

- 暂不支持多人协作 / 实时共享（需后端支持，超出静态页面能力）。
- 暂不接入真实路径规划（如驾车导航路线），路段以"两点间大圆线"展示。
- 暂不提供多张路线切换 / 多份数据在同一页面管理；若需多条路线，可另建一个 HTML 页面或使用不同 JSON 文件。

## 5. 数据结构

导出的 JSON 文件结构如下：

```json
{
  "title": "我的旅行路线",
  "points": [
    {
      "id": "p_x3k29d",
      "name": "北京",
      "lat": 39.9042,
      "lng": 116.4074,
      "transport": "flight"
    },
    {
      "id": "p_a7b1cd",
      "name": "上海",
      "lat": 31.2304,
      "lng": 121.4737,
      "transport": "transit"
    }
  ]
}
```

字段说明：

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `title` | string | 否 | 路线标题，默认 "我的旅行路线" |
| `points[].id` | string | 否 | 本地唯一标识，缺省时自动生成 |
| `points[].name` | string | 否 | 途经点名称，缺省时自动填充 |
| `points[].lat` | number | 是 | 纬度（WGS84），范围 -90 ~ 90 |
| `points[].lng` | number | 是 | 经度（WGS84），范围 -180 ~ 180 |
| `points[].transport` | string | 否 | 交通方式枚举：`flight / transit / drive / walk / train / ship`，缺省为 `drive` |

**渲染约定**：将 `points` 按下标顺序依次连接，最后一段为 `points[last] → points[0]`，形成闭环。

## 6. 技术与依赖

- 地图库：Leaflet 1.9.4（unpkg CDN）
- 瓦片服务：OpenStreetMap（免费，要求署名）
- 本地存储：浏览器 `localStorage`，键 `travel-route-data-v1`
- 距离计算：Haversine 公式（大圆距离，单位 km）
- 无构建步骤：纯静态 HTML / CSS / JS，与 GitHub Pages 原生兼容

### 6.1 部署步骤

1. 将 `travel-route.html` 上传至 GitHub Pages 仓库根目录（或其它子目录均可）。
2. 在仓库中创建目录 `data/`，把导出的 JSON 命名为 `data/travel-route.json` 提交到仓库；
   页面打开时会自动按 R4.3 的优先级尝试加载此文件。
3. 若需要分享给他人查看"指定路线"，可以使用 URL：
   `https://<username>.github.io/<repo>/travel-route.html?data=data/my-trip.json`
4. 提交并推送后，GitHub Pages 将在数分钟内更新生效。

## 7. 交互流程图（简化）

```
打开页面
 └─> 加载本地缓存 / 默认演示数据
      └─> 地图视图自动 fitBounds
用户操作循环：
 ├─> 点击「添加途经点」→ 进入添加模式 → 点击地图 → 填写名称 → 渲染新点 + 闭环
 ├─> 在某段交通方式下拉中切换 → 路线颜色/线型立即更新
 ├─> 拖动 Marker 到新位置 → 坐标与路段重新计算
 ├─> 点击上移 / 下移 → 调整路线顺序
 ├─> 点击删除 → 确认 → 重新闭环
 ├─> 点击「导出 JSON」→ 下载 .json → 用户手动提交到仓库
 └─> 点击「导入 JSON」→ 选择文件 → 校验 → 加载展示
```

## 8. 边界与错误处理

- 当 `points.length < 2` 时不绘制任何路段，并在侧栏显示"空状态"提示。
- 导入 JSON 缺少 `lat/lng` 或格式非法时，弹窗提示具体错误位置，不覆盖当前数据。
- `localStorage` 不可用时静默降级，只在内存中保留当前会话数据。
- 首次加载时若本地无任何数据，展示一条包含 4 个示例点的示例路线，以便用户快速理解。

## 9. 后续可扩展方向

- 为每个途经点增加日期、备注字段，并以时间轴方式展示。
- 使用 Leaflet 插件（如 `leaflet-routing-machine`）接入真实道路导航。
- 支持在 `data/` 目录下发现多个 JSON 文件，以下拉菜单切换不同路线。
- 提供"一键生成提交到仓库的 PR / issue body"快捷入口（需要 GitHub Token）。
